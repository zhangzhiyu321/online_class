# 开发模式使用说明

## 如何查看页面效果

我已经为你开启了**开发模式**，现在你可以**不需要登录**就能查看所有页面的显示效果了！

### 方法一：通过首页导航（推荐）

1. 启动项目：
```bash
npm run dev
```

2. 访问首页：http://localhost:5173
   - 首页会显示所有页面的导航卡片
   - 点击任意卡片即可跳转到对应页面

### 方法二：直接访问页面路径

直接在浏览器地址栏输入以下路径：

- **教师列表**：http://localhost:5173/teachers
- **教师详情**：http://localhost:5173/teacher/1
- **创建预约**：http://localhost:5173/appointment/create/1
- **预约列表**：http://localhost:5173/appointments
- **预约详情**：http://localhost:5173/appointment/1
- **支付列表**：http://localhost:5173/payments
- **支付详情**：http://localhost:5173/payment/1
- **聊天列表**：http://localhost:5173/chats
- **聊天窗口**：http://localhost:5173/chat/1
- **个人中心**：http://localhost:5173/profile
- **登录页面**：http://localhost:5173/login
- **注册页面**：http://localhost:5173/register

## 开发模式配置

开发模式的开关在 `src/router/index.js` 文件中：

```javascript
// 开发模式：设置为 true 可以跳过登录验证，方便查看页面效果
const DEV_MODE = true // 开发时设为 true，生产环境请设为 false
```

### 注意事项

1. **开发模式已开启**：当前 `DEV_MODE = true`，所有页面都可以直接访问
2. **生产环境**：部署到生产环境时，请将 `DEV_MODE` 改为 `false`
3. **数据展示**：由于没有后端数据，部分页面可能显示"暂无数据"或加载失败，这是正常的
4. **API 调用**：如果页面调用了 API 接口，可能会在控制台看到错误，不影响页面显示

## 页面功能说明

### 可以正常查看的页面

✅ **登录/注册页面** - 完整的表单和样式
✅ **教师列表页面** - 卡片布局、筛选功能（UI 完整）
✅ **教师详情页面** - 信息展示布局（可能无数据）
✅ **预约相关页面** - 表单和列表布局（可能无数据）
✅ **支付相关页面** - 列表和详情布局（可能无数据）
✅ **聊天相关页面** - 聊天界面布局（可能无数据）
✅ **个人中心页面** - 表单布局（可能无数据）

### 可能遇到的问题

1. **API 错误**：如果页面调用了 API，控制台可能会显示错误，这是正常的
2. **空数据**：列表页面可能显示"暂无数据"，这是正常的
3. **加载状态**：部分页面可能一直显示加载中，因为无法获取数据

## 关闭开发模式

当你需要测试真实的登录流程时，可以：

1. 打开 `src/router/index.js`
2. 将 `DEV_MODE` 改为 `false`
3. 保存文件，刷新浏览器
4. 现在访问需要登录的页面会跳转到登录页

## 提示

- 所有页面的 UI 和布局都是完整的
- 响应式设计在移动端和桌面端都能正常显示
- 可以通过浏览器的开发者工具切换设备视图来测试响应式效果
